                      # patch函数内流程

                      1.第一步：判断旧节点是否为一个vnode，不是则创建一个vnode

                      2.第二步：判断新旧节点是否为同一节点，不是则用新节点替换节点

                      3.第三步：如果是同一节点，则进行精细化比较

                      # 判断新旧节点是否为同一节点

                      key相同且sel相同时，如果用的text且变化了则更新旧节点的innerText值，如果不是则对他们的children进行diff算法即updateChildren方法

                      如果不为同一节点，则创建新节点并插入到旧节点之前，然后将节点移除

                      # diff算法思想

                      设置四个指针：新前、旧前、新后、旧后，按照以下方式去匹配是否为同一虚拟节点，若满足条件，则新前和旧前+1，新后与旧后--

                      循环体中要跳过oldCh与newCh中为空的项，具体做法就是验证新前、旧前、新后与旧后是否存在，不存在则上移或下移，然后再else if下面判断

                      1.新前与旧前相等，新前与旧前下移

                      2.新后与旧后相等，新后与旧后上移

                      3.新后与旧前（命中时，将旧前的节点移动到旧后之后），新后上移，旧前下移

                      4.新前与旧后（命中时，将旧后的节点移动到旧前之前），新前下移，旧后上移

                      5.若都没有命中，则在旧节点中寻找与新前相同的节点

                      如果能找到，则将新前patchVnode给找到的节点并将其移动到旧前之前，再在旧节点中将匹配的这项设置为undefined

                      如果没找到，则将新前插入到旧前之前

                      最后将新前下移

                      6.结束循环之后，若newCh中有剩余，则将创建剩余的新节点并插入到旧前之前；若oldCh中有剩余，则将剩余的节点移除
