                    # async和await的使用指南

                    # 前言

                    最近读《JavaScript高级程序设计 第四版》的时候，受到书中关于Promise与async的讲解， 于是便在工作使用了一番Promise+async来解决异步问题，并由此总结出一些内容，特在此处记录记下来， 供自己参考。

                    # 大致内容

                    1. Promise的特性
                    2. Promise的价值
                    3. async函数的特性
                    4. await表达式的特性
                    5. async/await的价值

                    # 以下是本文中用到的术语

                    1. pendingP (resolvedP resolvedP)
                    // 处于pending的Promise实例
Promise[[pending]];
// resolved rejectedP同理
                    1. executor(resolve, reject)
                    new Promise<T>(executor(resolve, reject)); // 传入Promise构造函数的回调函数
                    1. onResolved onRejected
                    // Promise实例then方法中的参数
Promise.resolve(1).then(onResolved, onRejected);
                    1. 值：resolvedP中内部的值
                    2. 理由：rejectedP中内部的值

                    # 1. Promise的特性

                    # 实例状态

                    • 三个状态：pending resolved rejected
                    • 不可逆性：从pendingP转变为resolvedP或rejectedP后不能再改变状态

                    # 执行器函数

                    • 立即执行：创建pendingP时就会执行executor函数
                    • 转变状态：在executor中调用resolve或reject会使pendingP变为resolveP或rejectedP
                    • 不被捕获：executor在执行中抛出的异常不会被new Promise()代码外层的try-catch捕获

                    # 静态方法

                    • 幂等性：当x为一个Promise实例且f为Promise.resolve时，f(f(x)) === f(x) === x。

                    # 处理程序

                    • 互斥性：最多只会执行定义在同一个then中的onResolved与onRejected中的一个
                    • 返回Promise：onResolved与onRejected的返回值会被Promise.resolve包装后作为then的返回值
                    • 抛出异常：onResolved或onRejected中抛出异常时,then方法会返回带有这个异常的rejectedP
                    • 微任务：onResolved或onRejected会在Promise实例状态为对应状态时被推入任务队列
                    • 顺序执行：多个then中定义的处理程序会按照其定义的顺序执行
                    • 值传递性：若在一个then中没有定义相关的处理程序，则Promise实例内部的值或理由会被传给Promise.resolve或Promise.reject并作为then的返回值

                    # 值缓存

                    • 值缓存性：Promise实例不管处理哪种状态，都可以在任意时刻被.then并在处理程序中取得相应的值或理由

                    # 2. Promise的价值

                    1. 串行：执行多个异步操作，并且下一步依赖上一步的结果
                    type PromiseFunc<T, V> = (singleArg: T) => Promise<V>;

function composePromise<T, V>(promiseFuncs: Array<PromiseFunc<any, any>>): PromiseFunc<T, V> {
    return (singleArg: T) => {
        return promiseFuncs.reduce((promiseP: Promise<any>, promiseFunc) => promiseP.then(promiseFunc), Promise.resolve(singleArg));
    };
}
                    1. 取多个结果：同时拿到多个异步的结果，并且不用关心这些异步在何时完成
                    Promise.all(promises).then(reses => {
    // ...
});
                    1. 并行：同时开始执行多个异步操作，并且这些异步操作使用相同的参数
                    type PromiseFunc<T, V> = (singleArg: T) => Promise<V>;

function concurrentPromise<T>(promiseFuncs: Array<PromiseFunc<any, any>>): PromiseFunc<T, any[]> {
    return (singleArg: T) => {
        const promises = promiseFuncs.map((promiseFunc) => promiseFunc(singleArg));
        return Promise.all(promises);
    };
}
                    1. 存储异步：使用promise存储一个异步操作与结果，并可以在任意位置以及任意时刻通过.then获取结果

                    实例：从服务端获取加密公钥，并对所有接口的请求体进行加密

                    方案: 将从服务端获取加密公钥这步的操作与结果缓存在一个promise中，并将原有的公共的post请求方法放在此promise的onResolved处理函数中，即可在所有接口调用前获取到公钥

                    // 存储获取加密公钥
const fetchEncryptKey = fetch('/api/encryptKey').then(response => response.text());
// 公共的post请求方法
const publicPostFunc = (params) => fetch(params.url, params.options).then(onResolved);
// 加密请求体的post请求方法
const encryptedPostFunc = (params) => fetchEncryptKey.then(encryptKey => {
    // 将params中的部分内容进行加密...
    const encryptedParams = encryptParams(params);
    return publicPostFunc(encryptedParams);
});

                    # 3. async函数的特性

                    1. 返回Promise: async函数永远返回Promise实例
                    2. resolve包装：async函数内return的值会被Promise.resolve()包装并作为async函数的返回值
                    3. 抛出异常：async函数内部若抛出错误，则这个错误将被Promise.reject()方法包装并作为async函数的返回值

                    # 4. await表达式的特性

                    1. 等待：await会使后续的代码变为微任务并在await表达式取得值后推入任务队列
                    2. 包装：await后的值会被Promise.resolve包装
                    3. 解包：await resolvedP时会将结果作为表达式的值
                    4. 释放：await rejectedP时会throw rejectedP中的理由

                    # 5. async/await的价值

                    1. 接收异步操作的值
                    const res = await fetch('...');
                    1. 异步任务同步化
                    // 执行一个异步操作
await doSomeAsyncFunc();
// 异步完成后，接着做一些处理
doSomeSyncFunc();
                    1. async版串行
                    type PromiseFunc<T, V> = (singleArg: T) => Promise<V>;

function asynComposePromise<T, V>(promiseFuncs: Array<PromiseFunc<any, any>>): PromiseFunc<T, V> {
    return async (singleArg: T) => {
        let res: any = singleArg;
        for (const promiseFunc of promiseFuncs) {
            res = await promiseFunc(res);
        }
        return res as V;
    };
}

                    # 6. 异常处理

                    1. 代码分层

                      a. api层（定义接口路径、传入参数及类型、返回值类型等）

                      b. 服务层（调用api层代码并进行业务逻辑处理）

                      c. 事件处理程序（调用服务层或api层代码来处理用户事件、生命周期函数等）

                    2. 事件处理程序层处理

                    // api层
async function loginApiFunc(param: any): Promise<any> {
    return await fetch('...');
}

// service层
async function doLogin(params: any): Promise<any> {
    const res = await loginApiFunc(params);
    if (res.code === PASSWORD_WRONG) {
        // ...
    }
}

// 事件处理程序
async function handleLogin(): Promise<void> {
    try {
        const {username, password} = this.user;
        if (!username || !password) {
            throw '请输入用户名和密码';
        }
        this.loading();
        await doLogin({username, password});
        this.toast('登录成功');
    } catch (errMsg) {
        this.alert(errMsg);
    }
}

                    说明：因为在service或api层等相对事件处理层更底层的代码中抛出的错误会被await释放。

                    假设上面的例子在loginApiFunc中报错了抛出了错误err，那么在dologin方法中的await将释放err使dologin方法返回一个内部值为err的rejectedP，在 handleLogin方法内的await也会处释放这个错误err并最终被try-catch捕获到。

                    # 总结

                    总之，使用async来解决异步问题，是非常简单且清晰且明了的，相对于一些第三方异步库（如rxjs）而言，虽然async功能并不多，但async的优势在于原生支持、语法简单、使用方便。
